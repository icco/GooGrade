Account

An account is the foundation of our three user classes: Teacher, Teacher Assistant and Student.

It is where information all three of those user types originate, such as email addresses, their names and login usernames.

Course

A Course is an important storage class for a list of Students, Teacher Assistants, Grading Rules and other details about the course, such as the Course Title and Section

Importantly, it houses methods that manage the list of Students.

EmailAddress

The Email Address is a simple class that stores an Account's email address.

It differs from an ordinary String class in that it contains methods that ensure that the email address is valid, for instance, making sure that the String contains a @ sign.

TeacherAssistant

  • The most versatile usertype
  • Ranges from mere slave to almighty admin
  • Available features completly dependant on set permissions)

Announcement

The Announcement class represents the messages a Teacher wishes to display to his Students.

The class stores the fields for the Title and Body of the text the Teacher has entered.

In addition, it also will collect the date and time of modification and store it as well.

Email

The Email class contains all the fields needed for an Email to be sent out to Students in a course.

It stores the fields for the Title, Body and recipients of the email as entered in by a Teacher or Teacher Assistant.

Assignment

The Assignment class contains the information about a particular Assignemnt, including the points it is worth, the Due Date and its Description.

An Assignment can be one of several categories:Test, Quiz, Homework, Final, Participation, and Other.

File

Each instance of the File class represents one file uploaded by a user onto the system.

The class keeps tabs on who the file is from.

For instance, homework uploaded by a Student would be put up for download for a Teacher
